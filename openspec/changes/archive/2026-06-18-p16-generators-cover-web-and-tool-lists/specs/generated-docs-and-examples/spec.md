## MODIFIED Requirements

### Requirement: README presents exact predownloaded versions and ready-to-use examples

`readme.md` SHALL be code-generated by `docs/build.mjs` from
`config/version.json` and SHALL cover **every published Linux image**
(`flutter-android` and `flutter-web`). For each image it SHALL render the
per-image "Main tools" list required by the `image-tool-inventory` capability
(the README content contract), with every version sourced from
`config/version.json` — including Java (OpenJDK) from `android.java.version`. The
README SHALL include, per image, the registry badges and a Running Containers
table, a usage snippet whose image tag equals the manifest Flutter version, and
inline GitHub Actions / GitLab CI examples. It SHALL name all four supported CI
platforms (GitHub Actions, GitLab CI, Gitea, Forgejo) and link to `examples/`,
and SHALL include a generated table of contents. The same generated `readme.md`
SHALL serve both the GitHub README and the Docker Hub description.

The experience context is a CI engineer reading only the README who compares
`flutter-android` and `flutter-web` by their exact shipped tool versions and
copies a working usage snippet — without the README drifting from the manifest.

> MODIFICATION (from the p15 single-image requirement): the generated README now
> covers two images, renders the per-image Main-tools lists defined by
> `image-tool-inventory`, and sources Java from `android.java.version`. The
> generator (`docs/build.mjs`) is the implementation of `image-tool-inventory`'s
> content contract.

#### Scenario: Both images are presented from the manifest

- **WHEN** `readme.md` is generated
- **THEN** it contains a `flutter-android` section and a `flutter-web` section
- **AND** each shows that image's Main-tools list with versions from
  `config/version.json` (the `image-tool-inventory` contract)

#### Scenario: Usage snippets use the current tag

- **WHEN** `readme.md` is generated for manifest Flutter version `X.Y.Z`
- **THEN** the flutter-android and flutter-web usage snippets reference their
  images at tag `X.Y.Z`

#### Scenario: All four platforms are named

- **WHEN** a reader views `readme.md`
- **THEN** GitHub Actions, GitLab CI, Gitea, and Forgejo are all named
- **AND** a link to the `examples/` directory is present

#### Scenario: Generated README reproduces the committed README

- **GIVEN** the committed `readme.md`
- **WHEN** `mise run docs` regenerates it from `config/version.json`
- **THEN** there is no diff (the in-sync CI gate passes)
